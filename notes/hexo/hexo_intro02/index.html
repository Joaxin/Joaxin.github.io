Butterfly 使用了 data files的特性，用于平滑升级，所以得把主题的配置文件_config.yml复制到工作目录下，比如source/_data/butterfly.yml下，之后编辑butterfly.yml即可简单配置主题

TOC为文章目录，设置可为：toc : true， toc_number: true, 其中auto_open决定每次打开文章时，是否会自动打开目录。

copyright用于显示文章版权模块 (默认为设置中 post_copyright 的 enable 配置)，如果有文章（例如：转载文章）不需要显示版权，可以在文章 Front-matter 单独设置copyright: false。

post 页的顶部图会优先显示各自 front-matter 中的 top_img, 如果没有设置，则会缩略图（即各自 front-matter 中的 cover)，cover为文章封面，如果没有则会显示显示默认的 cover，否则就显示 default_top_img 图片，默认在butterfly.yml中：

description中为文章摘要，因为之前的主题用的是<!-- more --> ,这点烧汤有不同，默认截取的为butterfly.yml的配置，但不推荐，因为会断片。

在 butterfly 里，有三种可供选择

  1. description 只显示 description
  2. both 优先选择 description，如果没有配置 description，则显示自动节选的内容
  3. auto_excerpt 只显示自动节选

主页的顶部图可在主题配置文件中index_img设置, archives 页的顶部图可以在主题配置文件中设置 archive_img, 其他可在各自的page md页面的top_img中设置(url 或者 其他), 页面如果没有设置各自的 top_img，则会显示default_top_img 图片。

yml 文件中 的tag_img 和 category_img 分别是 子标签页 和 子分类 的 顶部图配置，指 xxx/tags/android 這種子頁面的top_img。

主页会显示纯颜色的顶部图, 其他 page 的顶部图没有设置时，也会显示纯颜色的顶部图

Anchor - 文章锚点

开启文章锚点后，当你在文章页进行滚动时，文章链接会根据标题 ID 进行替换
(注意：每替换一次，会留下一个歷史记录。所以如果一篇文章有很多锚点的话，网页的歷史记录会很多。)

Outdated - 文章过期提醒

可设置是否显示文章过期提醒，以更新时间为基准。

  • limit_day： 距离更新时间多少天才显示文章过期提醒

  • message_prev ： 天数之前的文字

  • message_next：天数之后的文字
