Windows 系统目录解析
Created2018-10-17|Updated2019-12-13|SystemWindows
|Word count:2k|Reading time:6min|Post View:|

文件夹

  • Intel
    相关的软件 驱动活动日志（安装主板芯片组驱动留下的安装日志文件）
    Intel/ExtremeGraphics:Intel 显卡相关文件
    Intel./logs:intel 相关日志活动

  • Perflogs
    系统日志信息文件夹，如磁盘扫描 错误信息，是系统自动生成的，系统测试之后，测试记录文件也会自动存放在这个文件夹下。可以删掉，但不建议，因为这样反而降低系统速度。

  • Users

    用来储存系统所有用户数据、用户应用程序数据、桌面和开始菜单等内容的文件夹， 比如\Users\Administrator 共享给\Documents and Settings\Administrator, 实质是mklink软(符号)链接。

  • Program Files: 存储 64位的应用程序文件关联文件

  • Program Files (86):存储 32 位的应用程序文件关联文件

  • $Recycle.Bin

    回收站文件，删除后会自动重建，用于保存删除文件数据。

  • RECYCLER : 回收站

  • Boot

    用于保存操作系统引导文件的文件夹。

  • Documents and Settings

    系统用户设置文件夹，包括各个用户的文档、收藏夹、上网浏览信息、配置文件等。这里面保存着系统所有用户的文档和账户设置，尤其是里面的default user、all users、administrator和以你当前登陆用户名的文件夹。

  • Program Data

    系统系统文件夹，放置程序的使用数据、设置等文件，不建议删除

  • Recovery

    用于存放恢复系统用的恢复环境的文件夹，如果您习惯于系统出现状况后重装系统的话，可以删除这个文件夹，对系统不会造成任何影响。

  • System Volume Information

    这是任何一个分区下都会存在的一个文件夹，一般翻译成系统卷标信息文件夹。用于存储系统还原的备份信息以及磁盘分区数据信息。

文件

  • bootmgr

    系统启动引导文件,一旦删除，系统将不能启动

  • config.sys

    用于设置一些相关的系统环境,如命令提示符等，该文件内存储着一些配置命令

  • hiberfil.sys

    系统的休眠功能所占用的系统盘磁盘空间文件

    关闭：powercfg -h off

  • pagefile.sys

    虚拟内存页面文件

  • swapfile.sys

    Win8/Win8.1或者Win10系统Metro应用所使用的交换文件，是重要的受保护的系统文件。

Windows

  • Addins ：系统附加文件夹，用来存放系统附加功能的文件

  • AppCompat ：应用程序软件

  • AppPatch：应用程序修补备份文件夹，用来存储应用程序的修补文件

  • Assembly ：.net 程序安装后留有的支持文件, 软件的程序集名称 公钥标记 处理器等信息

  • Boot （引导）：系统启动的引导文件。一键还原等软件的文件夹

  • Branding ：品牌

  • CSC：Command Scheduling Chain,命令调度链 ; 此文件主要用客户机保障，用于通信网络的，命令调度指挥分配的。

  • *Common Signal Channel, 公共信号信道 ; 此文件删除可能引起网络和计算运行不稳定的情况

  • *Communication Systems Center, 通信系统 ; 你可以进行反汇编将其命令改变， 可以不依赖微软的通信系统。

  • Cursors ： 鼠标指针文件夹，扩展的鼠标文件

  • Debug：系统调试文件夹，用来存放系统运行过程中调试模块的日志文件

  • Diagnostics ：WINDOWS 系统诊断工具，出问题了自动运行。

  • DigitalLocker ：数字锁

  • Downloaded Program Files ：下载程序文件夹

  • en-US：语言文件

  • Fonts: 字体文件夹。要安装某种字体只需将字体文件复制到该目录下即可

  • Globalization: 全球化，全球化涉及识别必须得到支持的区域设置，设计支持这些区域设置的功能，以及编写在任何受支持的区域设置中都同样很好地运行的代码。

  • Help ：Windows 帮助文件

  • IME ：用来存放系统输入法文件

  • Inf ：用来存放 INF 文件。 INF 文件最常见的应用是为硬件设备提供驱动程序服务，不建议删除其中文件。

  • Installer: Windows Installer 服务的缓存文件夹，用于防止MSI应用程序安装包的缓存数据文件，系统中安装的msi安装格式的程序越多，它的体积就越大,可使用该软件清理。

  • L2Schemas ： 操作系统文件的各种通信设置 /配置

  • LiveKernelReports ：操作系统内核报告存放文件

  • Logs ：日志文件

  • Media ：声音文件夹，开关机等 wav 文件存放于此

  • Microsoft.NET ：提供构建 WEB 服务器的应用程序模型和关键技术。

  • Migration： Windows migration临时文件夹

  • Minidump： 计算机每次意外停止时 Windows 都会新建一个文件

  • ModemLogs ：调制解调器的日志信息

  • *Offline Web Pages ：脱机浏览文件的存放位置。当某个站点被设成允许脱机使用时，就会在该文件夹中生成对应的文件

  • Panther：日志文件说明。包含有关安装过程中的安装操作的信息。

  • Performance：存放系统评估工具 WINSAT

  • PLA ：扩展标记语言，一种简单的数据存储语言。

  • PolicyDefinitions ：双语对照查看

  • Prefetch：预读取文件夹，用来存放系统已访问过的文件的预读信息 (此信息是系统在访问时自动生成的新信息， 以加快文件的访问速度， 其扩展名为 “PF”。可以将此文件夹中的文件删除。

  • Registration ：注册文件夹，用来存放用于系统 COM ＋或者其他组件注册的相关文件

  • RemotePackages：远程文件

  • Rescache：资源缓存文件夹

  • Resources：系统资源文件夹，用来存放系统 在桌面主题

  • SchCache：系统文件夹，好像跟 Active Directory 服务有关

  • Schemas：xp sp3 新的主题文件夹

  • Security ：系统安全文件夹，用来存放系统重要的数据文件

  • ServiceProfiles ：windows 终端服务配置文件

  • Servicing ：微软对驱动程序的数字签名驱动服务相关文件夹

  • Setup：安装文件，删了不能升级

  • ShellNew ：起的是模板的作用 ,当你选择 “新建 ”命令中 ,该命令就将到此目录中寻找对应的模板文件 ,如果你将这些文件删除 ,“新建 ”命令就名存实亡了。

  • SoftwareDistribution ：主要是用来存放 WINDOWS UPDATE 时的更新日志及相关补丁下载的文件。理论上是可以删除的。

  • Speech：语言文件夹

  • System：存放一些系统需要的动态链接库和驱动程序

  • system32：存放 Windows 的系统文件和硬件驱动程序。

  • SysWOW64: 是一个 Windows 操作系统的子系统 , 能够运行 32-bit 应用 windows 操作系统程序 , 并且在所有的 64-bit 版本的 windows 上都存在

  • TAPI: Telephony Application Programming Interface。是一套用来编写与电信业务相关的程序的函数。

  • Tasks：是任务管理器相关文件

  • Temp 系统临时文件夹，其中内容可以全部删

  • Tracing：是 Windows Live 的文件 ,运行 Windows Live 就会出现

  • twain_32 扫描仪相关

  • Vss：软件测试的相关文档，测试代码、自动化测试程序代码均需要作为配置项，纳入到配置管理库中

  • Web：网络上的文件夹，就好象某些网站所提供的网络硬盘的功能一样，通过资源管理器，就可以把你的资料存放在网络服务器上， 比如壁纸

  • Winsxs: 存储各个版本的 Windows XP 组件，减少因为 DLL 文件而引起的配置问题, 占空间大户，可用Dism++ 清理

  • zh-CN ：通常是用来存放简体中文的本地化信息
