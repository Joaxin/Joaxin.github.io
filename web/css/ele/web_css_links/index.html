Links

When styling links, it is important to understand how to make use of pseudo-classes to style link states effectively, and how to style links for use in common varied interface features such as navigation menus and tabs. We‚Äôll look at all these topics in this article.

Link states

The first thing to understand is the concept of link states ‚Äî different states that links can exist in, which can be styled using different pseudo-classes:

  • Link (unvisited): The default state that defines the style for normal unvisited links. This can be specifically styled using the :link pseudo class.
  • Visited: A link when it has already been visited (exists in the browser‚Äôs history), defines the style for visited links, styled using the :visited pseudo class.
  • Hover: a link is hovered when the mouse is over, styled using the :hover pseudo class.
  • Focus: A link when it has been focused (for example moved to by a keyboard user using the Tab key or similar, or programmatically focused using HTMLElement.focus()) ‚Äî this is styled using the :focus pseudo class.
  • Active: A link when it is being activated (e.g. clicked on), styled using the :activepseudo class.

You‚Äôll notice a few things as you explore the default styles:

  • Links are underlined.
  • Unvisited links are blue.
  • Visited links are purple.
  • Hovering a link makes the mouse pointer change to a little hand icon.
  • Focused links have an outline around them ‚Äî you should be able to focus on the links on this page with the keyboard by pressing the tab key (On Mac, you may need enable the Full Keyboard Access: All controls option by pressing Ctrl + F7 before this will work.)
  • Active links are red (Try holding down the mouse button on the link as you click it.)

Note: All the links in the examples in this page are fake links ‚Äî a # (hash, or pound sign) is put in place of the real URL. This is because if the real links were included, clicking on them would break the examples (you‚Äôd end up with an error, or a page loaded in the embedded example that you couldn‚Äôt get back from.)

The order is important because the link styles build on one another, for example the styles in the first rule will apply to all the subsequent ones, and when a link is being activated, it is also being hovered over. If you put these in the wrong order, things won‚Äôt work properly. To remember the order, you could try using a mnemonic like LoVe Fears HAte.

A common practice is to include icons on links to provide more of an indicator as to what kind of content the link points to. Let‚Äôs look at a really simple example that adds an icon to external links (links that lead to other sites.) The last rule however is interesting ‚Äî here we are inserting a custom background image on external links in a similar manner to how we handled custom bullets on list items in the last article ‚Äî this time however we are using background shorthand instead of the individual properties. We set the path to the image we want to insert, specify no-repeat so we only get one copy inserted, and then specify the position as 100% of the way over to the right of the text content, and 0 pixels from the top.

Finally, we set some padding-right on the links to make space for the background image to appear in, so we aren‚Äôt overlapping it with the text.

A final word ‚Äî how did we select just external links? The text ‚Äúhttp‚Äù should therefore only appear in external links, and we can select this with an attribute selector: a[href*="http"] selects `` elements, but only if they have an href attribute with a value that contains ‚Äúhttp‚Äù somewhere inside it.

Styling links as buttons

In addition, links are quite commonly styled to look and behave like buttons in certain circumstances ‚Äî a website navigation menu is usually marked up as a list containing links, and this can be easily styled to look like a set of control buttons or tabs that provide the user with access to other parts of the site. Let‚Äôs explain what‚Äôs going on here, focusing on the most interesting parts:

  • Our second rule removes the default padding from the ul element, and sets its width to span 100% of the outer container

  • li elements are normally block by default (see types of CSS boxes for a refresher), meaning that they will sit on their own lines.     • As in previous examples, we start by turning off the default text-decoration and outline ‚Äî we don‚Äôt want those spoiling our look.

  • Next, we set the display to inline-block ‚Äî `` elements are inline by default and, while we don‚Äôt want them to spill onto their own lines like a value of blockwould achieve, we do want to be able to size them. inline-block allows us to do this.

    • Now onto the sizing! We want to fill up the whole width of the ``, leave a little margin between each button (but not a gap at the right hand edge), and we have 5 buttons to accommodate that should all be the same size. To do this, we set the width to 19.5%, and the margin-right to 0.625%. You‚Äôll notice that all this width adds up to 100.625%, which would make the last button overflow the <ul> and fall down to the next line. Note: You may have noticed that the list items in the HTML are all put on the same line as each other ‚Äî this is done because spaces/line breaks in between inline block elements create spaces on the page, just like the spaces in between words, and such spaces would break our horizontal navigation menu layout.
