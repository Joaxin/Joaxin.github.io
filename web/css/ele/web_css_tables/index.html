A typical HTML table

Let’s start by looking at a typical HTML table. The most important parts to note are as follows:

  • A table-layout specifies how the width of table columns is calculated, the value of fixed is generally a good idea to set on your table, as it makes the table behave a bit more predictably by default. The entire column width follows the width of its heading, making for a nice way to size your table columns. The table-layout . The possible values are:
    auto and fixed

    We’ve coupled this with a width of 100%, meaning that the table will fill any container it is put in, and be nicely responsive (although it would still need some more work to get it looking good on narrow screen widths).

  • The border-collapse property specifies whether the table borders are collapsed into a single border or separated as default. If the borders are separate, the border-spacing property can be used to change the spacing.

  • The caption-side property specifies the position of a table caption. The values can be set as top or bottom.

  • The empty-cells property specifies whether or not to display borders and background on empty cells (<td></td>), show and hide
